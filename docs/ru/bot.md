# Документация Бота

## Обзор

Telegram бот с интеграцией ИИ-агента и функциональностью напоминаний.

## Команды

### /start
Инициализирует бота и показывает приветственное сообщение.

**Использование**: `/start`

**Ответ**: "Бот запущен"

### /help
Показывает справку по всем командам.

**Использование**: `/help` или `/help <команда>`

**Ответ**: Полный текст справки или справка по конкретной команде.

**Примеры**:
- `/help` - Показать все команды
- `/help add` - Показать справку по команде add
- `/help reminders` - Показать справку по команде reminders

### /add
Добавляет новое напоминание.

**Использование**: `/add <имя> <период> <время> <день>`

**Параметры**:
- `имя`: Имя напоминания (строка)
- `период`: Один из: daily, weekly, monthly, yearly
- `время`: Время в формате ЧЧ:ММ (например, "14:30")
- `день`: Зависит от периода:
  - daily: опустить или пустое
  - weekly: 1-7 (1=Понедельник, 7=Воскресенье)
  - monthly: 1-31 (день месяца)
  - yearly: ДД.ММ (например, "25.12" для Рождества)

**Примеры**:
```
/add тренировка daily 08:00
/add встреча weekly 10:00 1
/add счет monthly 01:00 15
/add деньрождения yearly 09:00 25.12
```

**Валидация**:
- Время должно быть валидным ЧЧ:ММ
- Формат дня зависит от периода
- Период должен быть одним из разрешенных значений

### /reminders
Показывает все напоминания пользователя.

**Использование**: `/reminders`

**Ответ**: Список напоминаний со статусом:
```
Ваши напоминания:
- тренировка (daily, 08:00) - активно
- встреча (weekly, 10:00, день: 1) - активно
```

### /deactivate
Деактивирует напоминание по имени.

**Использование**: `/deactivate <имя>`

**Ответ**: Подтверждение или сообщение "не найдено".

### /del
Удаляет напоминание по имени.

**Использование**: `/del <имя>`

**Ответ**: Подтверждение или сообщение "не найдено".

## Обработка Сообщений

Любое не-командное сообщение отправляется ИИ-агенту.

**Обработка**:
1. Создает/переиспользует HTTP-сессию для пользователя
2. Отправляет запрос в API backend
3. Валидирует ответ
4. Возвращает ответ или сообщение об ошибке

**Сообщения об Ошибках** (на русском):
- "ошибка в ответе" - Неверный формат ответа
- "пустой ответ" - Пустой ответ
- "API error {статус}: {сообщение}" - HTTP ошибки
- "Request timed out after 200 seconds." - Таймаут
- "Unexpected error: {исключение}" - Другие исключения

## Фоновые Функции

### Управление Сессиями
- Поддерживает до 50 HTTP-сессий на пользователя
- Сессии переиспользуются между сообщениями
- Самая старая сессия удаляется при достижении лимита
- Сессии закрываются при выключении бота

### Воркер Напоминаний
- Работает каждую минуту в фоне
- Проверяет базу данных SQLite на просроченные напоминания
- Отправляет сообщения с напоминаниями пользователям
- Поддерживает все типы периодов с правильным сопоставлением даты/времени

## Технические Детали

- Построен с aiogram (асинхронный фреймворк Telegram)
- Использует aiohttp для HTTP вызовов с таймаутом 200с
- База данных SQLite для напоминаний
- Переменная окружения: `BOT_TOKEN`
