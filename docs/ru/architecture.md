# Архитектура

## Обзор

Проект Agent - это Telegram бот с ИИ-агентом, который предоставляет диалоговые ответы и функциональность напоминаний. Система построена с использованием асинхронного Python с FastAPI, aiogram и aiohttp.

## Компоненты

### 1. Telegram Бот (`bot/`)

- **bot.py**: Основное приложение бота с использованием aiogram
  - Обрабатывает сообщения пользователей и команды
  - Управляет пулом HTTP-сессий на пользователя (макс. 50 сессий)
  - Интегрирует функциональность напоминаний
  - Запускает фоновый воркер напоминаний

- **client.py**: Обертка HTTP-клиента
  - Управляет сессиями aiohttp на пользователя
  - Обрабатывает API-запросы с таймаутом (200с)
  - Валидирует ответы и обрабатывает ошибки
  - Предоставляет понятные сообщения об ошибках на русском

- **help_generator.py**: Генератор текста справки
  - Генерирует форматированные сообщения справки для команд бота
  - Поддерживает общую справку и справку по конкретным командам
  - Предоставляет информацию по устранению неполадок

### 2. API Backend (`api/`)

- **main.py**: Приложение FastAPI
  - Единственный эндпоинт `/run` для запросов агента
  - Принимает JSON с `user_id` и `text`
  - Возвращает JSON с полем `answer`
  - Полностью асинхронный

### 3. Ядро Агента (`agent/`)

- **core.py**: Логика агента
  - Строит промпты из пользовательского ввода
  - Асинхронно вызывает LLM
  - Простой шаблон промпта

- **llm.py**: LLM-клиент
  - Использует aiohttp для вызова API Ollama
  - Настраиваемая модель (llama3.1:latest)
  - Обрабатывает таймауты и ошибки
  - Возвращает чистый текстовый ответ

### 4. Сервисы (`services/`)

- **reminders.py**: Движок напоминаний
  - SQLite база данных для постоянного хранения
  - Фоновый воркер проверяющий напоминания каждую минуту
  - Поддерживает периоды ежедневно/еженедельно/ежемесячно/ежегодно
  - Отправляет напоминания через бота пользователям

## Поток Данных

1. **Сообщение Пользователя** → Telegram Бот
2. **Бот** → HTTP Клиент → API Backend
3. **API** → Ядро Агента → LLM Клиент → Ollama
4. **Ответ** ← ← ← ← ← ← ← ← ← ←
5. **Бот** → Пользователь

## Асинхронный Дизайн

- Все сетевые вызовы используют aiohttp
- Бот использует асинхронный фреймворк aiogram
- Фоновые задачи для напоминаний
- Пул сессий для эффективности

## База Данных

- SQLite файл: `reminders.db`
- Таблица: `reminders`
  - id, user_id, name, period, reminder_time, day, active

## Конфигурация

- Переменные окружения через `.env`
- BOT_TOKEN: Токен Telegram бота
- URL Ollama захардкожен (localhost:11434)

## Обработка Ошибок

- API ошибки: HTTP статус + сообщение
- Таймауты: 200с для API, 300с для LLM
- Валидация: Проверка формата ответа
- Понятные сообщения об ошибках на русском
